# Quick Start

## Installation

**1) Clone the repository:**

```
cd <local-git-path>
git clone https://wwwin-github.cisco.com/AS-Customer/sdwanac.git sdwanac
```

**2) Create the sdwanac docker image:**

```
cd <local-git-path>/sdwanac/docker
docker build -t sdwanac .
```

The docker container image includes all required software packages to validate the input data, generate the configurations, and deploy them via the vManage API.

**3) Launch the sdwanac docker container:**

```
docker run \
-dit \
--name sdwanac \
-v <local-git-path>/sdwanac:/root/sdwanac \
sdwanac
```

## Usage

### Configuration Deployment Steps

**1) Attach to the docker container**

```
docker attach sdwanac
```

**2) Validate the data input files and merge them into a single file called "sdwan.yml"**

```
cd ~/sdwanac
iac-validate -s schema/sdwan.yml -r validation/rules/ data/ -o sdwan.yml
```

Notes:

* All YAML dictionaries in the data folder must be rooted on "sdwan" and follow the structure defined in the sdwan YAML schema
* `iac-validate` performs the following actions:
  * YAML syntax validation of the input files, using `yamllint`
  * YAML semantic validation of the input files, based on the SDWAN-as-code schema
  * Merge the input data files into a single YAML data file ()
  * Semantic validation of the merged data using the validation rules (in `~/sdwanac/validation/rules/`)
* The merged YAML data is written to `~/sdwanac/sdwan.yml`, which is the input file used to generate the configurations and deploy them to the vManage

**3) Update the encrypted strings in the input data** (required when using a different vManage)

Generate the encrypted strings with your vManage, and update them in the input data (all parameters with vManage-encrypted strings, identified by `$CRYPT_CLUSTER$` value).

```
cd ~/sdwanac/python
python3 sdwan-encrypt-string.py -s <clear-text-string>
```

**4) Validate the data input files and merge them into a single file called "sdwan.yml"** (if the encrypted strings have been updated)

```
cd ~/sdwanac
iac-validate -s schema/sdwan.yml -r validation/rules/ data/ -o sdwan.yml
```

**5) Generate the target configurations**

```
cd ~/sdwanac/ansible
ansible-playbook generate_templates.yml
```

**6) Deploy the target configurations**

```
cd ~/sdwanac/python
python3 sdwan-deploy-templates.py
```

### Input Data

The **input data** is provided via YAML files, which are used to generate the target configurations. The input data is maintained in a separate git repository.

**Example Input Data**

An example input data repository can be found in [sdwanac-example](https://wwwin-github.cisco.com/ialonso/sdwanac-example).

The example repository includes the templates for the supported features and options.

The example repository can be used to deploy configurations to an SD-WAN setup in the CX EMEA NFVI-PODs:

### Important Notes

#### Encrypted Strings

Passwords, keys and shared secrets in the data model must be provided using either a hash value or an encrypted string (as used in the vManage template payloads).

The SDWAN-as-Code repository includes a python script to generate the hash values and the encrypted forms of a clear-text string.

```
docker attach sdwanac
cd ~/sdwanac/python
python3 sdwan-encrypt-string.py -s <clear-text-string>
```

Example:


NOTE: **the vManage-encrypted string is specific to each vManage**, i.e., it can be used only in the same vManage where it has been generated.

#### Feature Template Variables

In the definition of the feature templates in the data model, for the parameters where per-device variables are supported, the variable name is included in the parameter value using this convention: `DEVICE_VARIABLE;<variable-name>`.

#### Per-Device Variables

The per-device variables are provided in the "sites" section of the data model. The per-device variables is a flat list of `variable-name: variable-value` entries for each SD-WAN router.

The repository includes a python script (`~/sdwanac/python/sdwan-list-device-template-variables.py`) to print the list of variables for each device template. The script parses the feature templates and device templates, and provides the list of variables per device template.

The repository also includes a validation rule to verify that all required variables have been provided for each SD-WAN router (Rule 301).
